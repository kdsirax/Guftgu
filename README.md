# 💬 Guftgu - Realtime Chat & WebRTC Video Calling Application

Guftgu is a premium, state-of-the-art real-time private messaging and peer-to-peer audio/video calling web application. Built using React, Sass, and Firebase, Guftgu offers an exceptionally polished chat experience including rich aesthetics, real-time online status trackers, interactive image sharing, and direct WebRTC connections.

---

## ✨ Features

* **🔐 Diverse Authentication Options**:
  * Register and log in using standard Email and Password configurations.
  * Instant access via **Sign in with Google** integration.
  * Optional profile pictures with default fallback avatars generated dynamically using the DiceBear API.
* **🟢 Real-Time Presence & Status**:
  * Sidebar renders all registered platform users in real-time.
  * Real-time **online presence tracker** showing a green dot badge next to the user's name when they are active.
  * Automatic transition to offline status upon explicit logout or browser/tab closure.
* **💬 Seamless Chat & Multimedia Sharing**:
  * Real-time message threads using Firebase Firestore subscriptions.
  * Share images instantly with a floating interactive **thumbnail preview** to review or cancel selected media before hitting send.
  * Last-sent preview text formatting (e.g., showing `📷 Image`).
* **📹 Peer-to-Peer WebRTC Video Calls**:
  * High-definition video and audio streaming direct from browser to browser.
  * Built using pure browser WebRTC APIs (`RTCPeerConnection`) with free, public Google STUN servers.
  * **Calling overlays**:
    * **Ringing Notification Card**: Centered glassmorphic card presenting Accept (Green) and Decline (Red) options with pulsing rings.
    * **Calling Screen**: Outgoing pulsing animation while waiting for an answer.
    * **Active Calling Overlay**: Full-screen remote video stream with a rounded floating local Picture-in-Picture window.
    * **Call Controls**: Instant microphone mute toggle, camera toggle, and hang up.

## 🛠️ Tech Stack

* **Frontend**: React (v18), Sass, Vanilla CSS
* **Database & Signaling**: Firebase Firestore
* **Storage**: Firebase Storage (for user avatars and chat attachments)
* **Auth Provider**: Firebase Authentication (Email/Password + Google Provider)
* **Networking Protocol**: WebRTC (P2P Audio/Video Streams)

---

## 📂 Firestore Collection Schema

The application structures data across four root collections in Firestore:

1. **`users`**: User profile credentials and presence indicators.
   * `uid`, `displayName`, `email`, `photoURL`, `isOnline`
2. **`userChats`**: Meta record mapping all chats a user is involved in.
   * Keyed by user ID, mapping to subobjects containing `chatId`, `userInfo`, `lastMessage`, and `date`.
3. **`chats`**: Stores message history arrays for room channels.
   * `messages`: Array of items containing `id`, `text`, `senderId`, `date`, and optional `img` link.
4. **`calls`**: Manages WebRTC signaling handshakes.
   * `id`, `callerId`, `callerName`, `callerPhoto`, `receiverId`, `receiverName`, `receiverPhoto`, `status`, `offer`, `answer`, `createdAt`
   * **Subcollections**:
     * `callerCandidates`: Set of ICE candidates generated by the calling peer.
     * `receiverCandidates`: Set of ICE candidates generated by the receiving peer.

## 🚀 Getting Started

### 1. Prerequisites
Ensure you have Node.js (v16+) and npm installed.

### 2. Configuration Setup
Create a `.env` file in the root of the project and add your Firebase API key:
```env
REACT_APP_FIREBASE_KEY="YOUR_FIREBASE_API_KEY"
```

Also verify the credentials inside `src/firebase.js` match your project config.

### 3. Installation
Install the project dependencies:
```bash
npm install
```

### 4. Running Locally
Launch the local React development server:
```bash
npm start
```
Open [http://localhost:3000](http://localhost:3000) in your browser.

## Recommended Security Rules

### Firestore Rules
```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null;
    }
    match /chats/{chatId} {
      allow read, write: if request.auth != null;
      match /{allChildren=**} {
        allow read, write: if request.auth != null;
      }
    }
    match /userChats/{userId} {
      allow read, write: if request.auth != null;
    }
    match /calls/{callId} {
      allow read, write: if request.auth != null;
      match /{allChildren=**} {
        allow read, write: if request.auth != null;
      }
    }
  }
}
```

### Firebase Storage Rules
```javascript
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /{allPaths=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```
